# 数据集划分方案说明

本文档说明本项目天气数据来源、天气数据处理方式，以及短期预测和长期预测任务的数据集划分方案。

## 1. 天气数据来源

课程 PDF 要求融合法国月度气象数据，来源为：

- [Données climatologiques de base - mensuelles](https://www.data.gouv.fr/fr/datasets/donnees-climatologiques-de-base-mensuelles)

该数据集由 Météo-France 发布，包含法国本土和海外气象站自建站以来的月度基础气候数据。data.gouv 页面说明数据以省份和时间段拆分为压缩 CSV 文件，并提供所有可用气象参数。

本项目用电数据来自法国 Sceaux 附近住户，因此选择 Hauts-de-Seine 省，即法国省编号 `92`。项目下载并使用以下官方资源：

- `MENS_departement_92_periode_1950-2024`
- 原始文件：`data/raw/weather/MENSQ_92_previous-1950-2024.csv.gz`
- 字段说明：`data/raw/weather/MENSQ_descriptif_champs.csv`

该文件覆盖 1950-01 至 2024-12，能够完整覆盖用电数据的 2006-12 至 2010-11 时间范围。

## 2. 天气字段处理

课程要求使用以下天气字段：

| 字段 | 含义 | 处理方式 |
| --- | --- | --- |
| `RR` | 月累计降水高度 | 按课程要求除以 10 |
| `NBJRR1` | 当月日降水 >= 1 mm 的天数 | 保留月度数值 |
| `NBJRR5` | 当月日降水 >= 5 mm 的天数 | 保留月度数值 |
| `NBJRR10` | 当月日降水 >= 10 mm 的天数 | 保留月度数值 |
| `NBJBROU` | 当月雾出现天数 | 保留月度数值 |

官方 92 省文件中包含多个气象站。考虑到单个站点在 2006-2010 期间可能存在字段缺失，项目采用如下策略：

1. 读取 92 省所有站点月度记录。
2. 对每个月、每个天气字段，在所有有值站点之间取平均值。
3. 对聚合后仍缺失的字段使用按时间顺序插值，并做前后向填充。
4. 将月度天气特征按月份合并到每日用电数据。

处理后的天气文件为：

- `data/processed/weather_monthly_normalized.csv`

天气合并后的每日数据为：

- `data/processed/daily_power_weather.csv`

合并后每日数据包含 13 个可建模特征：

1. `global_active_power`
2. `global_reactive_power`
3. `sub_metering_1`
4. `sub_metering_2`
5. `sub_metering_3`
6. `sub_metering_remainder`
7. `voltage`
8. `global_intensity`
9. `RR`
10. `NBJRR1`
11. `NBJRR5`
12. `NBJRR10`
13. `NBJBROU`

## 3. 每日用电数据范围

原始分钟级用电数据经过缺失值处理和天级聚合后，得到：

- 起始日期：2006-12-17
- 结束日期：2010-11-26
- 日数据行数：1441 天
- 缺失值：合并天气后无残留缺失值

预测目标为每日 `global_active_power`。

## 4. 窗口构造规则

课程要求基于过去 90 天预测未来曲线。因此每个样本由输入窗口和输出窗口组成：

```text
输入 X：连续 90 天的多变量特征
输出 y：未来 90 天或未来 365 天的 global_active_power
```

项目使用滑动窗口构造样本，步长为 1 天。

短期预测任务：

```text
X shape: [样本数, 90, 13]
y shape: [样本数, 90]
```

长期预测任务：

```text
X shape: [样本数, 90, 13]
y shape: [样本数, 365]
```

## 5. 划分原则

时间序列预测不能随机打乱样本，否则会让未来信息泄漏到训练阶段。因此本项目按时间顺序划分：

- 训练集：前 70% 窗口样本；
- 验证集：中间 15% 窗口样本；
- 测试集：最后 15% 窗口样本。

特征标准化也只在训练段上拟合，然后应用到验证集和测试集，避免测试集统计信息泄漏。

由于课程 PDF 中提到的 `train.csv` 和 `tes.csv/test.csv` 当前未在项目目录中提供，本项目采用上述时间顺序划分。若后续获得官方划分文件，应优先使用官方划分，并重新生成样本。

## 8. 为什么短期和长期任务分开划分

短期任务和长期任务的输出长度不同，因此可构造的样本总数不同：

- 90 天预测需要 `90 + 90 = 180` 天；
- 365 天预测需要 `90 + 365 = 455` 天。

长期任务需要更长的未来标签，所以可用窗口更少。两个任务应分别生成样本、分别训练模型、分别评估，符合课程要求中“长期预测模型参数不能用于短期预测”的要求。

## 9. 后续实验使用方式

训练短期模型时使用：

```text
data/processed/short_horizon_samples.npz
```

训练长期模型时使用：

```text
data/processed/long_horizon_samples.npz
```

每个文件内部已经包含：

- `x_train`, `y_train`
- `x_val`, `y_val`
- `x_test`, `y_test`
- 对应日期范围
- 特征名
- 输入长度
- 预测长度
- 标准化参数

训练时不得重新打乱时间划分，只能在训练集内部按 batch shuffle。
